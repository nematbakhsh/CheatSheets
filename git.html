Git

Version Control System (VCS) for tracking changes in computer files

Installing Git

  • Linux

    (Debian) sudo apt-get install git

    (Fedora) sudo yum install git

  • Mac and Windows

    http://git-scm.com

    • Configuration

    • git config --global user.name STRING username
    • git config --global user.email STRING email

Basic Commands

git init
Initializes local git repository
git add <file>
Add files to staging area
git rm --cached <file>
Unstage a file
git status
Check status
git commit
Commit files in staging area to local respository
git push
Push to remote respository
git pull
Pull latest from remote repository
git clone <url>
Clone repository into a new directory

Other commands

git --version
Returns git version
git commit -m STRING comment
Commit staged files

Information given by git status

On branch <branchname>
Shows the current working branch
Untracked files
New files which are not staged
Changes to be committed
New or modified files which are staged
Changes not staged for commit
Staged files which contain chenges that are not staged

.gitignore file

This file should be put in root directory (same as .git folder) and each line contains the name of the file/folder that should be ignored by git add .

Example: Inside of .gitignore 

			log.txt
			/node_modules
			*.jpg

Branching

git branch <branchname>
Create new branch
git checkout <branchname>
Switch branches
git merge <branchname>
Merges specified branch to the current branch

Remote repository

echo String content >> README.md
Create a README file
git remote add origin <url>
Add the remote repository which is named origin
git remote
Returns name of the remote repository
git push -u origin master
Pushes the master branch to the remote repository (runs for the first time)
git push
Pushes the master branch to the remote repository
