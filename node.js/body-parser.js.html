Node.js Body-Parser

Parse incoming request bodies in a middleware before your handlers, available under the req.body property. (without it the req.body would be undefined )

Installation

npm install --save body-parser

Usage

Add to index.js
// body-parser API
const bodyParser = require('body-parser')
//
// parse application/x-www-form-urlencoded
app.use(bodyParser.urlencoded({ 
    extended : false // does not have a default value, therefore should be passed
}))
//
// parse application/json
app.use(bodyParser.json())

Body-Parser Methods

bodyParser.json([options]) Returns middleware that only parses json and only looks at requests where the Content-Type header matches the type option. (Here default type value is "application/json") bodyParser.text([options]) Returns middleware that parses all bodies as a string and only looks at requests where the Content-Type header matches the type option. (Here default type value is "text/plain") bodyParser.urlencoded([options]) Returns middleware that only parses urlencoded bodies and only looks at requests where the Content-Type header matches the type option. (Here default type value is "application/x-www-form-urlencoded") bodyParser.raw([options]) Returns middleware that parses all bodies as a Buffer and only looks at requests where the Content-Type header matches the type option. (Here default type value is "application/octet-stream")

Body-Parser Methods Options

inflate limit type verify extended (only for urlencoded method)allows to choose between parsing the URL-encoded data with the querystring library (when false) or the qs library (when true)
