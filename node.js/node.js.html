NPM

Where is it installed: By default, npm is installed alongside node in C:\Program Files (x86)\nodejs

npm's globally installed packages (including, potentially, npm itself) are stored separately in a user-specific directory (which is currently

C:\Users\<username>\AppData\Roaming\npm).

Because the installer puts

C:\Program Files (x86)\nodejs

before

C:\Users\<username>\AppData\Roaming\npm

on your PATH, it will always use the version of npm installed with node instead of the version of npm you installed using npm -g install npm@<version>.
node -v | node --version returns node.js version npm -v | npm --version returns npm version npm init initializes an app by creating a package.json file npm install Install all dependency and dev-dependency packages in package.json file npm install --production Install all dependency and packages in package.json file npm install (--save || --save-dev) <Package> Install a package (as dependency || dev-dependency) npm uninstall (--save || --save-dev) <Package> Uninstall a package npm install -g npm@latest installs the latest version of npm npm audit search for vulnerabilities) npm audit fix Scan your project for vulnerabilities and automatically install any compatible updates to vulnerable dependencies (use dangerous --force only if you know what you are doing) npm cache verify Verifies the contents of the cache folder, garbage collecting any unneeded data, and verifying the integrity of the cache index and all cached data. npm cache clean --force --force is a mandatory flag

package.json content

"name" : STRING "version" : STRING "description" : STRING "keywords" : Array array-of-strings "homepage" : STRING "bugs" : { "url" : STRING, "email" : STRING} "main" : STRING "scripts" : OBJECT "author" : STRING "license" : STRING "dependencies" : OBJECT "devDependencies" : OBJECT "repository" : { "type" : STRING, "url" : STRING "private" : true npm will refuse to publish private apps

In order to change default values of init use the set command

npm set init.author.name "<Your Name>" npm set init.author.email "you@example.com" npm set init.author.url "example.com" npm set init.license "MIT"

Path module

const path = require('path') calls the module path.basename(STRING filepath) returns only the base filename with extension path.dirname(STRING filepath) returns only the directory path.extname(STRING filepath) return file extension with dot path.parse(STRING filepath) rturns path object (an object with keys: root, dir, base, ext and name) path.join(...paths) concatenates paths path.sep Provides the platform-specific path segment separator: '\' for windows and '/' for POSIX path.resolve([...paths]) The given sequence of paths is processed from right to left, with each subsequent path prepended until an absolute path is constructed

Tip: path.resolve(__dirname, '..') returns the parent directory
