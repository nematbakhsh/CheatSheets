Dangerous file extension naming for includes

Plot

index.php 
require "inc/db.inc"; //which contains database setting

Now, if someone goes to ~/inc/db.inc, then Apache will render this file as a plain text and reveals its content. But if "db.inc" was named "db.php" then it would be rendered as php file and not revealing any php code.

Solution

Name your include and sensitive files properly, ike "db.php" and avoid "db.inc", etc.
